Software Engineer, Performance Investigation team, ECS

Location: St. Petersburg, Russia

Dell EMC ECS is the industry-leading Object Storage, designed for both traditional and next-generation workloads with unmatched scalability, consistency guarantees, flexibility, and resiliency. Scale-out and geo-distributed, the ECS object platform offers superior performance for apps that demand massive unstructured data storage capacity and global access, with the storage efficiency that only gets better when more sites are connected, and with all the enterprise-grade features that public cloud doesn’t provide.

As a member of the Performance Engineering team, you will be responsible for taking product code, instrumenting it for tracing, running traces, and analyzing the performance data to locate performance bottlenecks. After issues are identified, you will work with the component owner teams to develop and test solutions to the issues. Candidates must possess a passion for software performance tuning and strong communication skills to work cross-functionally with other teams.

A successful performance engineer will define and drive projects to measure, analyze, and improve the performance of the product over time. This will involve using existing tools and developing new ones to collect, store, analyze, and present performance-related data and metrics, as well as propose and prototype improvements. You will be part of a team that designs, builds, and utilizes these tools and processes, testing in large-scale cloud environments.

Key Responsibilities
  • Design and implement white-box performance testing of storage clusters, including initial setup of system configurations
  • Develop test automation
  • Update existing software to inject performance instrumentation code
  • Design, and execute test scenarios against various software versions
  • Analyze test results to isolate performance issues
  • Work cross-functionally with various component teams to resolve performance issues
  • Track component performance over time to identify performance regressions
Essential Requirements
  • Understanding of system and software architecture
  • Familiarity with Linux operating system concepts
  • Experience working with bug tracking systems
  • Understanding of the complex multi-threading s/w systems design and maintenance
  • Proficient in communication in English, good oral and written communication skills
  • Strong motivation to learn new things and embrace change
  • Good time management skills, ability to multi-task, team player
Desirable Requirements
  • Familiarity with Linux system IO and kernel tuning
  • Familiarity with performance analysis tools
  • Familiarity with Docker and Kubernetes
  • Familiarity with Flink, Spark and Kafka a strong plus
  • Experience in performance testing
  • Experience working in an Agile team
  • Understanding of object storage (e.g. AWS S3)
  • Experience working in remote virtual environments
  • Familiarity and experience working within CI environments (e.g. Jenkins)
  • Experience working in a globally distributed organization in a collaboration with other remote teams
