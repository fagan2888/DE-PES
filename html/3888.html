PURPOSE AND OBJECTIVES

The SAP HANA & Analytics Database organization is a global organization dedicated to delivering database technology and data management solutions that addresses customers' unique and competitive business requirements. The SAP HANA team develops a database that performs parallel in-memory processing of huge data sets to offer extremely fast real-time responses for analytic queries. Besides its analytical advantages, the transactional capabilities of SAP HANA enhance the product, turning it into a general-purpose database, not only optimized for SAP applications. The database also provides libraries for predictive, planning, text processing, spatial, graph, documents and business analytics – all on the same architecture.

To Offer This Cutting-edge Technology To Even More People And Enable The Database For Even More Use-cases, SAP HANA Is Not Only Available On Premise But Also In The Cloud Run By SAP

EXPECTATIONS AND TASKS

SAP HANA Cloud brings the power and performance of SAP HANA to the cloud. It enables the customer to manage their data storage, federation, and run powerful applications all within a single cloud solution. As software engineer for SAP HANA Cloud you will have an end-to-end responsibility of HANA Cloud scenarios.

On a usual day you will...
  • Define the use case that we want to address and implement
  • Build proof of concepts and prototypes of the defined use case
  • Take care all gaps within the use cases will be addressed and become part of the backlog of the development teams
  • Work together with development teams to discuss problems and their solutions
